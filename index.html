display 触发 transition

Dec 2 2013

之前对transition一直模棱两可：

当定义了transition后，任何用js改变的符合变换的样式都会执行动画

好吧，但原来是隐藏的时候（display:none），然后显示出来（display:block），就不会执行了，需要一些技巧。

一. 元素A默认是隐藏，要求显示出来，并有动画效果-透明度由0变为1

<script>
    function xxx() {
        A.style.transition = 'all 2s linear';
        A.style.opacity = 0;
        A.style.display = 'block';
        setTimeou(function(){
            A.style.opacity = 1;
        }, 10);
    }
</script>

后面的动作放到计时器里，就好用了哦。。。

测试题

Jul 31 2013

一. 问：一个数组，求对此数组成员进行去重

答： 

Array.prototype.uniq = function() {
    var i = 0;
    while (i < this.length) {
        var j = i + 1;

        while (j < this.length) {
            if (this[i] === this[j]) {
                this.splice(j, 1);
            }
            else {
                j++;
            }
        }
        i++;
    }
}

结果：

> var arr = [ 'a', 'a', 1, 2, '1'];
> arr.uniq();
> arr
  [ 'a', 1, 2, '1' ]

nginx的rewrite的处理顺序

Apr 10 2013

nginx的rewrite指令会经常用到，这里探讨一下的它的执行顺序

首先说一下指令rewrite的语法配置：

语法：rewrite regex replacement [ flag ]

flag标记有：

  • last — 表示完成本次location的rewrite，使用新地址重新进行一轮匹配（浏览器地址不变）
  • break — 本条规则匹配完成后， 终止所有匹配，返回文件（浏览器地址不变）
  • redirect — 返回302临时重定向（浏览器地址变为后面的跳转后的地址）
  • permanent — 返回301永久重定向（浏览器地址变为后面的跳转后的地址）

好吧，我相信对于last和break，你已经懵了，没事，往下看！！！！

table元素设置position为relative

Jan 23 2013

有一个需求，table的表头th有两个箭头，一个紧挨着th的上面，一个紧挨着th的下面，点击箭头可以排序，但是th的高度不固定，会随着内容的换行高度变化。

有几种方式：

一. 给th设置position为relative

css2.1规范
写到
The effect of ‘position:relative’ on table-row-group, table-header-group,table-footer-group, table-row, table-column-group, table-column, table-cell, and table-caption elements is undefined.

然后浏览器实现的也不一样，ie系列、chrome（自己测试高版本的是支持的）支持，firefox直接无视了此style设置（看来firefox下display为block，inline-block可以，table-cell不可以）。

二. 给th里加一个div，用此div包裹箭头，并设置此div定位为相对定位

测试发现：两个th的内容，一个是一行，一个是两行，然后div的高度就不一样（为内容的高度），看来div的高度必须设成th的高度才行呀，好吧， 将div高度设为100%，坑爹不起作用。因为th的高度必须有值，100%才起作用，但此th的高度是自适应的肯定不能写死，好吧，此路不同

三. 用第二种方式，但需要用js先计算出th的最大高度，然后设所有th为此高度，好吧，终于可以了，虽然比较坑爹

参考文档：

  • http://www.w3.org/TR/CSS21/visuren.html#choose-position
  • http://w3help.org/zh-cn/causes/RM8024

javascript支持label

Oct 10 2012

今天在翻jslint的时候发现这一条：

JavaScript allows any statement to have a label, and labels have a separate name space. JSLint is more strict.
JSLint expects labels only on statements that interact with break: switch, while, do, and for. JSLint expects that labels will be distinct from vars and parameters.

这时看不懂了，这个label指的是什么。通过询问同事，发现原来javascript还支持这么个东西，竟然之前都不知道。
看下面的代码就知道了。

<script>
    outer:for (var i = 0; i < 3; i++) {

        alert(i);
        for (var j = 0; j < 1; j++) {

            break outer;
        }
    }
</script>

这个outer就是那个label，正常的break只会跳出上一层的循环，这样就会跳出这两层for循环，在别的语言里还有goto，嘿嘿，不过js就没有了。天杀的，一直怎么就没有看到过呢。。。

使用fiddler自动下载指定文件

Aug 30 2012

使用fiddler构建动态数据桩———用fiddler按目录 自动下载保存 浏览器请求的文件.

前段时间想构建一个json数据桩，需要从线上拉下json数据，懒得动手一个个复制，就研究了下fiddler，通过fiddler自动按照规则从指定线上落下数据，本地保存。

需要使用fiddler工具，如果没有fiddler，请先行下载。 fiddler下载地址

配置方式：

  1. 先打开fiddler， 然后点击 工具栏—》Rules —》Customize Rules…, 打开fiddler的配置脚本， 用项目目录里的CustomRules.js内容覆盖掉此脚本。

  2. 在脚本文件中把数据桩地址变为本地自己的存储地址 static var gs_localPath: String = ‘F:/testdata’;
    保存即可。

  3. 将脚本中的www.xxx.com 改为自己要下载的网址。

  4. 打开fiddler- 工具栏- rules 选中Data storage，打开浏览器操作想操作的网址，系统里的所请求的action就会自动下载到本地配置地址。
    如果在开发过程中发现只差一个文件，则可在fiddler的链接栏里选中link，右击，点击 Data storage即可只更新此link（可多选）

    Ps:只会更新状态码是200的请求（当然这是可配的）。
